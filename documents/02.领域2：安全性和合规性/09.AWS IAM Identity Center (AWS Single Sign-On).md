## AWS IAM Identity Center (AWS Single Sign-On)

AWS IAM Identity Center（之前称为 AWS Single Sign-On，SSO）是 AWS 提供的一项服务，允许用户在多个 AWS 账户和业务应用程序之间实现单点登录（SSO）。这使得用户可以使用一个单一的身份来访问多个 AWS 资源和应用程序，简化了身份管理和访问控制的复杂性。以下是关于 AWS IAM Identity Center 的详细信息：

### 1. **核心功能**

- **单点登录 (SSO)**：用户可以通过一次身份验证访问所有授权的 AWS 账户和应用程序，无需重复输入凭证。
- **集成应用程序**：支持与多种第三方应用程序（如 Salesforce、Box、Microsoft 365 等）的集成，方便用户管理访问权限。
- **基于角色的访问**：根据用户的角色定义访问权限，确保用户仅访问所需的资源。

### 2. **身份管理**

- **用户目录**：IAM Identity Center 提供用户目录功能，可以管理组织中的用户和组。
- **外部身份提供者**：支持将现有的身份提供者（如 Active Directory 或其他 SAML 兼容的身份提供者）集成，便于将现有身份管理系统与 AWS 资源结合。

### 3. **访问控制**

- **基于组的权限管理**：用户可以通过组来管理访问权限，简化权限分配和审计。
- **细粒度权限**：可以为不同的用户和组分配不同的访问权限，以满足安全和合规性要求。

### 4. **审计与合规**

- **活动跟踪**：通过 AWS CloudTrail 记录 IAM Identity Center 中的活动，支持审计和合规性检查。
- **权限评估**：可以审查和管理用户的权限，以确保符合组织的安全政策。

### 5. **多因素身份验证 (MFA)**

- **增强安全性**：支持配置多因素身份验证，增加访问安全性，保护用户账户免受未经授权的访问。

### 6. **用户体验**

- **用户门户**：IAM Identity Center 提供用户门户，用户可以通过一个简单的界面查看和访问他们的应用程序和 AWS 资源。
- **无缝体验**：用户只需登录一次即可访问所有集成的应用程序，提高用户体验和生产力。

### 7. **跨账户访问**

- **跨 AWS 账户的访问管理**：通过 IAM Identity Center，组织可以轻松管理和控制跨多个 AWS 账户的用户访问。

### 8. **自定义品牌**

- **定制用户门户**：允许用户根据组织的需求自定义门户界面，提供个性化的登录体验。

### 9. **多区域支持**

- **全球可用性**：AWS IAM Identity Center 可以在多个区域使用，支持全球分布的用户访问。

### 总结
AWS IAM Identity Center 提供了一种安全、灵活且易于管理的方式来实现单点登录，帮助组织简化身份管理和访问控制。通过集成现有身份提供者、提供基于角色的权限管理和增强的安全性，IAM Identity Center 支持组织在多个 AWS 账户和应用程序中高效管理用户访问，确保安全合规。
